### 4b. Automate Page (AI Import)

**Key files:**
- `src/screens/automate/AutomateScreen.jsx` — single-import 3-phase flow (Upload → Review → Done)
- `src/screens/automate/BatchImport.jsx` — batch 4-step UI (Select → Group → Process → Summary), exposed as `forwardRef`; parent calls `batchRef.current.resolveGroup(id, result)`
- `src/logic/automateImport.js` — shared pipeline: `buildReviewState`, `computeBreakdowns`, `validateReviewData`, `insertVoyage`, `autoSaveReceipt`
- `src/logic/batchGrouping.js` — `buildGroups(files)`, `movePage`, `splitPage`, `mergeWithPrevious`
- `src/components/DocumentImport.jsx` — `pdfToImagesFromBase64`, `compressToJpeg` (PDF limit: 20 pages, scale 1.5, JPEG 0.82)
- `electron/handlers/ai.js` — makes the Anthropic API call

**Single-import flow:**
1. User picks files → `dialog:openDocuments` IPC → returns `{ filename, data: base64, mimeType, size, mtimeMs }`
2. PDFs rasterized in renderer via pdfjs-dist; images compressed to JPEG
3. All page images sent to `window.api.aiExtract(images)` → `ai:extract` IPC
4. Main process calls Claude API, returns parsed JSON
5. `buildReviewState` maps AI output to form shape; uncertain fields get amber ⚠ highlight; RS codes + P3 rows dropped
6. Staff edits review form, then `validateReviewData` → `insertVoyage` → upserts berthing + service records
7. `autoSaveReceipt` calculates receipt and saves; `receipt:exportPDFBatch` writes PDF silently to `C:\ShipFees\receipts\`

**Batch grouping:** Samsung filename `YYYYMMDD_HHMMSS` parsed; photos within 12 seconds grouped as same receipt. Falls back to `mtimeMs` when filename unparseable. Files without usable timestamp each get their own group.

**Auto-insert vs Needs Review:** Group only auto-inserts when ALL three pass: zero uncertain fields, all codes matched, all required-field validations pass.

**Position normalization (`POSITION_MAP`):**
```
QUAY / POS1 / POS_1 / P1    → Quay
P2 / POS_2 / POS2           → P2
EN RADE / ENRADE / EN-RADE  → En Rade
EN RADE FREE / CONGESTION / CONG / EN RADE LIBRE → Congestion ($0, saved)
P3 / POS3 / POS_3           → FREE (filtered out, never billed)
```

### 4c. Login System

**Key files:**
- `src/screens/Login.jsx` — form, EN/AR toggle before login
- `src/screens/ChangePassword.jsx` — forced on `must_change_password = 1`
- `src/context/SessionContext.jsx` — `login()`, `logout()`, heartbeat every 60s
- `src/App.jsx` — routing: no session → Login; `must_change_password` → ChangePassword; else → MainApp
- `electron/database/handlers/auth.js` — `login()`, `changePassword()`, `logout()`

**Mechanics:**
- `bcrypt.compareSync(password, hash)` — cost factor 10
- Username lookup case-insensitive (`COLLATE NOCASE`)
- On success: updates `last_login`, `is_online = 1`, `last_seen`; loads `user_permissions` rows
- Session object: `{ id, username, full_name, role, language, must_change_password, permissions[], perm_storage, perm_automate, perm_cma }`
- Heartbeat: `users:heartbeat` IPC every 60 seconds → updates `last_seen`
- No JWT/token — all session state in React `useState`; no configurable idle expiry
- Error codes: `invalid_login`, `account_disabled`

**Screen-level access control (MainApp.jsx):**
- `storage`, `automate`, `cma`: `admin` role OR matching `perm_*` column = 1
- `audit_log`, `user_management`: `admin` only
- `staff_view`: `manager` only
- All other screens: any authenticated user

**Key files:**
- `src/screens/users/UserManagementScreen.jsx` — admin CRUD UI with inline EditUserDialog + ResetPasswordDialog
- `src/screens/users/ManagerStaffScreen.jsx` — manager read-only presence view, refreshes every 30s
- `electron/database/handlers/users.js` — all user CRUD + permission management

**Functions:** `getAll`, `create`, `update`, `resetPassword`, `setActive`, `getPermissions`, `setPermission`, `checkHasRecords`, `deleteUser`, `heartbeat`

**Roles:** `admin`, `user`, `manager` (role CHECK expanded via schema migration)

**Two-track permission system:**
- `user_permissions` table (key strings): `generate_cma_receipt`, `edit_others_records`, `access_tariff_editor`
- `users` table columns (integer flags): `perm_storage`, `perm_automate`, `perm_cma`
- `setPermission` dispatches to column UPDATE or table INSERT/DELETE based on `COLUMN_PERMS = ['perm_storage', 'perm_automate', 'perm_cma']`

**Guards:**
- Cannot disable own account
- Cannot disable last active admin
- Cannot delete a user who has berthing records (disable instead)
- Cannot delete/disable own admin account

**Presence status (from `last_seen`):**
- Online: active in last 3 minutes
- Idle: 3–30 minutes
- Offline: 30+ minutes or never logged in

## 6. AI Processing (How the Model Works)

**File:** `electron/handlers/ai.js`

**Extraction model:** `claude-sonnet-4-6`, `max_tokens: 4096`
**Test connection model:** `claude-haiku-4-5-20251001`, `max_tokens: 5`

**Input:** Array of `{ data: base64, mediaType }` — all pages sent in one request as vision content blocks alongside the text prompt.

**Prompt strategy:** One large hardcoded `EXTRACTION_PROMPT` string. Instructions cover two document formats (CAMALI-style vs Port bill), position inference rules, container size detection, overtime handling, code corrections, and RS code exclusion. Returns only JSON.

**Variables extracted:**
```json
{
  "voyage_number": "string | null",
  "vessel_name": "string | null",
  "vessel_type": "General Cargo | Container | RoRo | null",
  "flag": "string | null",
  "shipping_agent": "string | null",
  "loa": "number | null",
  "ata": "DD/MM/YYYY | null",
  "atd": "DD/MM/YYYY | null",
  "berthing": [{ "position": "string", "days": "number", "total": "number" }],
  "services": [{ "code": "string", "container_size": "20ft|40ft|null", "quantity": "number", "price_per_unit": "number", "line_total": "number" }],
  "uncertain_fields": ["string"]
}
```

**Confidence / uncertain flagging:**
- No numeric confidence scores — model self-reports field names in `uncertain_fields` array
- Prompt is explicit about when NOT to flag (successfully inferred position, empty overtime table, zero overtime, found a number and used it, etc.)
- Renderer: any field in `uncertain_fields` → amber border (`#F59E0B`) + ⚠ indicator
- Service lines from unknown codes also get `_uncertain: true`

**Post-extraction mapping in `buildReviewState` (automateImport.js):**
- RS-prefixed codes dropped
- P3/POS3/POS_3 berthing rows dropped
- Position aliases normalized via `POSITION_MAP`
- Dates: `DD/MM/YYYY` → `YYYY-MM-DDT00:00`
- Code fixes: `C321` → `C321-E`, `TRSTD` → `TR-STD`
- Currency stripped from price fields: `"$26.54"` → `26.54`

**Error codes returned to renderer:**
`no_api_key`, `invalid_api_key_format`, `invalid_api_key` (HTTP 401), `api_error_{status}`, `empty_response`, `invalid_json`, `network_error`

**No hardcoded confidence thresholds** — the model decides what to flag via prompt rules alone.

## 10. Receipt Calculation Summary (receiptCalc.js)

```
RS lines              → excluded entirely
OT lines (/-E$/ or OT/OVERT) → always regular lines regardless of is_taxable

berthingTotal = max(sum(feeAfterDiscount + maintenanceFee), applicableMinimum)
              - 35% RoRo discount if vesselType = RoRo

price         = berthingTotal + sum(regularLines)
fundable      = min(price × 0.035, $450)

GC branch:
  taxableSubtotal = sum(QUAIT + STAMP lines)
  rehabFee        = (taxableSubtotal - 2) × 0.03511111   [0 if fundable capped]
  totalTax        = ((0.11 × taxableSubtotal - 0.22) × 0.035) + 0.11 × taxableSubtotal
  fixedTotal      = AUTOM + BILLF
  containerTax    = $0

Container branch:
  taxableSubtotal = $0
  rehabFee        = $0
  totalTax        = $0
  fixedTotal      = AUTOM + BILLF + STAMP ($8)
  containerTax    = $0.22

freshAmount = price + fundable + rehabFee + fixedTotal + taxableSubtotal + totalTax + containerTax
finalPrice  = Math.ceil(freshAmount)   [any decimal → ceiling; whole numbers unchanged]
```

## 11. Berthing Fee Calculation Summary (berthingCalc.js)

```
lIndex = 1 + (loa > 75) + (loa > 125) + (loa > 175)   → range 1–4

Quay:    rawFee = (d1 × R_D1[lIndex] + d2 × R_D2[lIndex] + d3 × R_D3[lIndex]) × loa
P2:      rawFee = days × R_P2[lIndex] × loa
En Rade: rawFee = days × 1 × loa   (flat, no lIndex — changed 2026-06-18)
Congestion: rawFee = 0 (saved as $0 record)

Day tiers: D1 = min(days, 5), D2 = min(max(days-5,0), 10), D3 = max(days-15, 0)

discountFactor from vessel_categories (Lebanese 0.50, Ro-Ro 0.65, Military 0.00, etc.)
feeAfterDiscount = rawFee × discountFactor
maintenanceFee   = maintenance='Yes' ? 3 × days × loa : 0

Minimum applied at receipt time (receiptCalc.js), not per row.
```

## 12. Client / Server Architecture

Two modes controlled by `%APPDATA%\Ship Fees\config.json`:

**Server mode (default):** App runs local SQLite + starts Express on port 3001. All IPC handlers hit local DB.

**Client mode:** App skips SQLite entirely. Every IPC call is proxied by `electron/client.js` → `POST http://{serverUrl}/api/{...}` with `x-token` header. The server PC runs in server mode and handles the DB.

**Known gap:** `server.js` exposes endpoints for auth, berthing, container, gc, receipts, CMA, users, audit, settings. Storage endpoints are NOT in `server.js` — client-mode Storage calls will 404.
