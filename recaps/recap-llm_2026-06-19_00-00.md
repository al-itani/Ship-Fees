## CHANGES
CALC LOGIC (storageCalc.js): FREE_DAYS=9, GRAIN_FREE_DAYS=12. Vehicle: 3-day periods, escalating rates by size (small/car/big), daily cap = V_DAILY_CAP[size]*totalDays. VehicleTransit: $16/period flat. Container local: escalating C_RATES[20|40]. Container transit/export: periods*20 (same both sizes). GC/Iron: 9-day blocks, GC_RATES=[3,4,6] or IRON_BLOCK_RATES=[2.4,4,6] per ton. Grain: same as GC blocks but 50% discount, 12-day free period. Forced: flat daily rate (no free period), per-ton for weight types.

AUDIT: table_name='storage_records', action prefix '[STORAGE]', old_data null on INSERT, full old row on UPDATE/DELETE.

ROLES: all roles can calc/save/view. Admin+Manager can edit. Admin only can soft-delete. Enforced in StorageRecords.jsx JSX (canEdit/canDelete checks on session.role).

AGENTS: loaded via window.api.getAgents() (berthing handler, returns string[] of names).
