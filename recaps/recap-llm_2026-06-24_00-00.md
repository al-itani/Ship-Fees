## CHANGES
- BerthingForm: `form.position` + `form.days` removed from form state, moved to `lines[]` array. `breakdown` → `breakdowns[]`. Fee calc runs per-line. Save builds `commonPayload` (vessel meta) + per-line `buildLinePayload`. Edit mode: `editRecord` prop replaced by `editVoyageNumber` string; form loads via `berthingGetByVoyage` API call.
- Auth restore: localStorage key `rememberedUserId` stores user ID (number as string). On mount, `SessionContext` calls `authRestoreSession` which re-fetches user row, validates `is_active` and `must_change_password`, marks `is_online`. Clears key on logout.
- DB reset: deletes billing data only; users/rates/agents/categories tables untouched. Audit row written BEFORE the transaction so it survives even if the tx fails partway.
- CMA LBP: rate not yet confirmed in code — check `CMAScreen.jsx` for the hardcoded LBP/USD rate used.

## WARNINGS
- `db:reset` uses raw `DELETE` (not soft delete) by design — this is intentional for a factory reset. It wipes `audit_log` last. Do not add soft-delete here.
- BerthingForm now expects `editVoyageNumber` (string) not `editRecord` (object) — any other caller must be updated
- `berthing:getByVoyage` returns rows ORDER BY created_at ASC — edit mode maps rows by index; order matters for surplus-delete logic
- db:reset is NOT available in client mode (`C` flag guard in main.js)
- Full restart required (`npm run dev`) for all these changes — they touch main.js, preload.js, and handlers
