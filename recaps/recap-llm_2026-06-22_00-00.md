## TASK
Build a single-file browser app (`tariff_c.html`) that reads two raw Excel files, runs the Article 22 empty container storage fee calculation, shows an agency table, and generates a printable Port of Beirut receipt per agency.

## CONTEXT
The main app is an Electron+React desktop billing system for Port of Beirut. A parallel Electron module for Tariff C already exists (`electron/database/handlers/tariffC.js`, `src/screens/tariff-c/TariffCScreen.jsx`, `src/screens/tariff-c/TariffCReceiptPreview.jsx`) but reads a pre-calculated Excel file. This new standalone HTML file must compute from two raw source files: Full_Discharge_YYYY-MM.xlsx and Empty_Stocks_YYYY-MM.xlsx. The billing counter is stored in localStorage key `tariff_c_billing_counter` (starts at 0, increments on Close, not on Open).

## CHANGES
Single 901-line HTML file with embedded CSS and JS. SheetJS loaded from CDN (xlsx 0.18.5). Three screens: Upload, Results Table, Receipt Overlay.

**Column detection:** `findCol(headers, pats)` scans first 30 rows for recognizable header names (agent/agence/agency, categ/category, 20'/20ft/\b20\b, 40'/40ft/\b40\b, date, box no/box number/\bbox\b). Throws descriptive error if agency + at least one TEU column not found.

**Discharge parsing (`parseDischarge`):** Skips rows where Categ. column is empty/NaN. Groups teu20+teu40 sums per agency. Agency key = `normName()` (lowercased, strips SAL/SARL/LTD/CO, removes non-alphanumeric).

**Stocks parsing (`parseStocks`):** Skips rows where Box No. column is empty/NaN (total rows). Aggregates daily TEUs per (agency, dateStr). Handles Date as JS Date, Excel serial (XLSX.SSF.parse_date_code), or DD/MM/YYYY string. Stores first Box No. encountered per agency.

**Calculation (`calculate`):** A = teu20 + teu40×2. D = Math.round((A×7)/30). Daily fee tiers: ≤D→$0, D<x≤2D→(x−D)×1.5, x>2D→D×1.5+(x−2D)×3.0. storageAmount = sum of daily fees, Math.round to 1 decimal. Output month = discharge filename month+1 (year rolls over if month>12).

**Agency matching (`findMaster`):** Normalizes name and does exact then token-overlap match (≥0.5 score) against 63-entry master list. Unknown agencies trigger warning but calculation continues. Results sorted by master list order.

**Receipt totals:** fundable = min(storageAmount×0.035, 450) rounded to 2dp. finalPrice = Math.ceil(storageAmount + 4.00 + 0.22 + fundable). totalTax = 0.22 (always fixed). Two sections: EMPTY CONTAINERS (ECM line, storageAmount) and OTHERS (AUTOM $1, BILLF $1, STAMP $2). Grand totals row + final-row with tax/finalPrice/fundable.

**Print:** @media print hides everything except #receipt-overlay, A4 portrait, @page margin:0.

## WARNINGS
- The Electron module (`tariffC.js`) reads a DIFFERENT file format (pre-calculated, fixed column positions 1–8). The standalone HTML reads raw source files — these are entirely separate input formats.
- `closeReceipt()` increments billing counter AND marks row Done — do not call it without user intent (it's irreversible in the current session).
- Agency name fuzzy matching uses token overlap ≥0.5 — ambiguous short names (e.g. "MSC") rely on exact normName match; adding similar-named agencies to master list could cause mis-matches.
- `excelDateStr()` handles 3 date formats; if the Empty Stocks file uses a 4th format (e.g. MM/DD/YYYY), dates will not aggregate correctly and daily fees will be wrong.
- `normName()` strips "co." which could corrupt names containing "co" as part of a word — unlikely but watch for it.
