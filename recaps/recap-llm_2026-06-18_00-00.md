## CHANGES
All 36 DB-touching IPC channels now dispatch based on `C`. Key arg-shape mismatches resolved:

1. `auth:changePassword` — client.js takes `(userId, currentPassword, newPassword)`; IPC only passes `(userId, newPassword)`. Fix: pass `null` for currentPassword -> `clientHandlers.changePassword(userId, null, newPassword)`.

2. `container:saveSession` and `gc:saveSession` — IPC passes a single `data` object; client.js functions take `(voyageNumber, lines, savedBy)`. Fix: destructure on dispatch -> `clientHandlers.containerSaveSession(data.voyageNumber, data.lines, data.created_by)`.

3. `cma:getReport` — IPC receives args as `(year, month)` matching preload call order; client.js signature is `(month, year)`. Fix: swap on dispatch -> `clientHandlers.cmaGetReport(month, year)`. Server receives named body `{ month, year }` so wire order is irrelevant on the server side.

4. `users:delete` — client.js exports `usersDeleteUser` (not `usersDelete`). Fix: call `clientHandlers.usersDeleteUser(id)`.

5. `berthing:delete`, `users:update/resetPassword/setActive/setPermission` — client.js functions do not accept `opts` or `adminId` args. Fix: pass only the args client.js uses; local path still receives all original args.

Local-only handlers (never dispatched to client.js):
`auth:logout`, `receipt:exportPDF`, `receipt:exportPDFBatch`, `dialog:openDocuments`, `ai:extract`, `ai:testConnection`, `cma:exportExcel`, `users:heartbeat`, `app:getConfig`.

Audit handlers (`audit:getEntries`, `audit:getFilterOptions`, `audit:logImport`) kept local-only — client.js has no audit implementation and cannot be modified per constraints.

## WARNINGS
- `container:saveSession` and `gc:saveSession` destructure `data.voyageNumber`, `data.lines`, `data.created_by` — if renderer changes the shape of the data object, the client dispatch breaks silently.
- `users:heartbeat` is intentionally local-only — do not route to clientHandlers even if client.js gains a heartbeat function later; it must always hit the local DB for presence tracking.
- `cma:getReport` arg swap is intentional — do not "fix" it without re-checking both the preload call order and client.js signature.
- Audit log on a client machine reads local (likely empty) DB — admin should always access audit from the server machine.
