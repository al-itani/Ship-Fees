# LLM Recap — Prompt 2: Client/Server Mode Split
DATE: 2026-06-18
PROJECT: Ship Fees Desktop App
STACK: Electron + React + SQLite (better-sqlite3) + Express (server.js)

## TASK
Add a client/server mode split so one PC runs the DB (server mode) and others proxy all IPC calls over HTTP (client mode).

## CONTEXT
Prompt 1 added an Express HTTP API in `electron/server.js` that mirrors every IPC handler via POST routes with a shared `handle(res, fn)` wrapper and `x-token` auth header.
Prompt 2 builds on that by adding a config-driven mode flag and a full HTTP client layer.
The app always boots the same binary; mode is determined by `config.json` in AppData.

## FILES_CHANGED
- FILE: `electron/configStore.js`
  ACTION: created
  REASON: Singleton that reads `%APPDATA%\Ship Fees\config.json` once; returns `{ mode, serverUrl, token }`; falls back to `{ mode: 'server' }` if file absent or malformed.

- FILE: `electron/client.js`
  ACTION: created
  REASON: HTTP client mirroring every `server.js` endpoint as named async functions. Uses `apiCall(endpoint, body)` helper with `x-token` header. Called by main process in client mode instead of local handlers.

- FILE: `electron/main.js`
  ACTION: edited
  REASON: Imports `getConfig`; only starts Express server if `appConfig.mode !== 'client'`; adds `app:getConfig` IPC handler so renderer can read mode/serverUrl at runtime.

- FILE: `electron/server.js`
  ACTION: edited
  REASON: Fixed `/api/auth/login` — `authHandlers.login` returns `{ success, user }` envelope; old code passed the envelope as data; new code unwraps it and returns `{ success: true, data: result.user }`.

## CHANGES
- `configStore.js`: reads `app.getPath('userData')/config.json`; expected shape `{ mode: 'client'|'server', serverUrl: 'http://...', token: '...' }`
- `client.js`: `apiCall` POSTs to `serverUrl + endpoint`, attaches `x-token: token`, throws on `json.success === false`
- All domain functions in `client.js` pack args into the body shape `server.js` expects (e.g. `berthingUpdate(id, data)` → `{ id, data }`)
- `main.js`: conditional `require('./server').startServer()` now guarded by `appConfig.mode !== 'client'`
- `main.js`: new IPC `app:getConfig` returns `{ mode, serverUrl, token }` (token included — renderer should not expose this in DevTools in production)
- `server.js` login fix: manually unwraps handler result instead of using `handle()` wrapper
